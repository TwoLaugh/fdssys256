# Module: AI Service

## Purpose
Centralised layer for all LLM interactions. Handles model routing, prompt templates, response parsing, cost tracking, and failure recovery. No other module calls the Anthropic API directly.

## Dependencies
**None.** This module wraps an external API (Anthropic Claude). Other modules call into it.

## Data Model

### ai_call_log
```sql
CREATE TABLE ai_call_log (
    id              BIGSERIAL PRIMARY KEY,
    task_type       VARCHAR(50) NOT NULL,
    model_used      VARCHAR(50) NOT NULL,
    model_tier      VARCHAR(20) NOT NULL,     -- frontier/mid/cheap
    input_tokens    INTEGER NOT NULL,
    output_tokens   INTEGER NOT NULL,
    cost_pence      INTEGER NOT NULL,
    latency_ms      INTEGER NOT NULL,
    success         BOOLEAN NOT NULL,
    error_message   TEXT,
    prompt_hash     VARCHAR(64),
    created_at      TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_ai_call_log_task ON ai_call_log(task_type);
CREATE INDEX idx_ai_call_log_created ON ai_call_log(created_at DESC);
```

├── health/

Each template has:
- System message
- Placeholders: `{{user_profile}}`, `{{pantry_state}}`, etc.
- Output format specification (JSON schema)

## Service Interface

```java
public interface AiService {
    /**
     * Execute a prompt template with context, parse response into target type.
     */
    <T> T execute(AiTaskType taskType, Map<String, Object> context, Class<T> responseType);

    /**
     * Chat-style interaction (recipe suggestions, feedback).
     */
    AiChatResponse chat(AiTaskType taskType, Map<String, Object> context, String userMessage);

    /**
     * Usage stats.
     */
    AiUsageDto getUsage(YearMonth month);
}

public enum AiTaskType {
    RECIPE_SELECTION, PLAN_ASSEMBLY, PLAN_ADJUSTMENT,
    RECIPE_IMPORT, RECIPE_GENERATE, RECIPE_EVOLVE, RECIPE_SUGGEST_CHANGES,
    FEEDBACK_INTERPRET, PREFERENCE_MODEL_UPDATE,
    INGREDIENT_PARSE, INGREDIENT_MATCH_USDA,
    RECIPE_DISCOVERY,
    WEEKLY_REVIEW, MONTHLY_REVIEW,
    TESCO_PRODUCT_MATCH
}
```

## Failure Handling

| Failure | Strategy |
|---------|----------|
| API timeout | Retry once, then fail gracefully |
| Rate limit (429) | Exponential backoff + retry |
| API down (500/503) | Return error, features degrade (viewing still works, generation doesn't) |
| Unparseable response | Retry once with stricter prompt, then fail |
| Context too large | Truncate lowest-priority context, retry |

## Consumed By
Every module that needs AI: Planner, Recipe, Discovery, Feedback, Health, Grocery, Nutrition Engine.

None. This is a utility module.
